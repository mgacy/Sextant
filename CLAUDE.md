## Project Overview

Sextant is a structural Swift code search CLI tool (`sextant`) that parses Swift source files using swift-syntax, providing structured queries beyond what grep or LSP can offer. All output is JSON.

## Build & Test Commands

```bash
swift build                    # Build all targets
swift test                     # Run all tests
swift test --filter FileParserTests    # Run a specific test suite
swift test --filter "extractsTopLevelDeclarations"  # Run a single test
swift run sextant --help       # Run the CLI
```

Linting uses SwiftLint via mise:

```bash
mise run lint                  # Lint the code
```

## Architecture

Two targets: an executable (`sextant`) and a library (`SextantLib`).

### SextantLib

The library handles parsing and querying. **`FileParser` is the isolation boundary for swift-syntax** — no swift-syntax types escape the public API. Everything downstream operates on owned value types:

- **`FileParser`** — parses Swift source into `FileOverview` (tree of `Declaration` nodes). Source is parsed once via `Parser.parse(source:)` and the resulting syntax tree is used for both declaration extraction and `SourceLocationConverter` creation.
- **`FileScanner`** — walks directories collecting `.swift` files, excluding `.build/`, `checkouts/`, `.index-build/`.
- **`SymbolTable`** — in-memory index built from `FileOverview`s. Keyed by name for exact-match lookup with optional kind filter.
- **`StructuralQuery`** — regex-based enum case search against full serialized declarations, plus symbol-by-kind filtering.
- **`TypeReferenceFinder`** — internal `SyntaxVisitor` subclass that finds type-position references (inheritance, parameters, return types, type annotations, typealias targets, generic constraints) while skipping function bodies, variable initializers, computed property bodies, and import statements. Used by `FileParser+References`.
- **Models** (`FileOverview.swift`, `SymbolEntry.swift`, `ReferenceMatch.swift`, `ParseResult.swift`) — `Declaration`, `FileOverview`, `SymbolEntry`, `SymbolKind`, `ReferenceMatch`, `ReferencePosition` are `Codable`, `Equatable`, `Sendable`. `ParseResult`, `ParseFailure`, `ReferenceResult` are `Sendable` only.
- **Extensions** (`Collection+Utils.swift`) — `isNotEmpty` convenience on `Collection`.
- **`@OmitEmpty`** (`OmitEmpty.swift`) — custom property wrapper used on `Declaration` fields to omit empty arrays/strings from JSON output. Uses `EmptyInitializable` protocol + `KeyedEncodingContainer`/`KeyedDecodingContainer` overloads.

### sextant (CLI)

Uses swift-argument-parser with four subcommands:

- `overview <file>` — JSON dump of declarations in a single file
- `lookup <name> [--kind] [--path]` — symbol lookup across a directory
- `enum-cases <path> [--pattern]` — enum case search with optional regex
- `references <name> [--path] [--position]` — find declaration-level type references across a directory

`ScanAndParse.swift` is shared infrastructure used by `LookupCommand`, `EnumCasesCommand`, and `ReferencesCommand` — scans directories via `FileScanner`, parses via `FileParser`, logs parse failures to stderr. `JSONOutput.swift` provides the shared `printJSON` helper (compact JSON with sorted keys by default). `OutputOptions.swift` defines shared `--pretty` and `--absolute` flags used by all subcommands. `SymbolKind+ExpressibleByArgument.swift` and `ReferencePosition+ExpressibleByArgument.swift` bridge library types to ArgumentParser. `Version.swift` is auto-generated by the `swift-version-file-plugin`.

### Test Fixtures

Tests use `.swift` fixture files in `Tests/SextantTests/Fixtures/` loaded via `Bundle.module`. Test suites: `FileParserTests`, `StructuralQueryTests`, `SymbolTableTests`, `ReferenceFinderTests`, `CompactEncodingTests`, `ConcurrencyTests`, `ExtensionActorInitializerTests`, `FileScannerTests`, `FullDeclarationTests`.

## Key Dependencies

- **swift-syntax** (601+) — Swift source parsing (`SwiftParser`), syntax tree traversal (`SwiftSyntax`), and source location resolution
- **swift-argument-parser** (1.5+) — CLI framework
- **swift-docc-plugin** (1.4.3+) — documentation generation
- **swift-version-file-plugin** (0.2+) — generates `Version.swift` from package version

Platform: macOS 14+, Swift 6.2.
